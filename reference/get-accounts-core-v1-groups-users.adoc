= Liste die Benutzer einer Gruppe auf
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/groups/{group_id}/users`#

Gibt ein JSON-Array aller übereinstimmenden Ressourcen in der Sammlung zurück. Wenn in der Anfrage einzelne Felder angegeben werden, enthält das JSON-Array der Elemente die angeforderten Felder für jede übereinstimmende Ressource in der angegebenen Reihenfolge.

|===
| Name | Typ | Erforderlich | Beschreibung 


| Typ | Zeichenkette | Wahr  a| 
Medientyp der Ressource. Definierte Werte sind:

* "application/astra-user"




| Version | Zeichenkette | Wahr  a| 
Version der Ressource. Definierte Werte sind:

* "1,0"
* "1,1"
* "1,2"




| id | Zeichenkette | Wahr  a| 
Weltweit eindeutiger Bezeichner der Ressource. Definierte Werte sind:

* Entspricht dem UUIDv4-Schema




| Zustand | Zeichenkette | Wahr  a| 
JSON-Zeichenkette mit einem Wert, der den Betriebsstatus des Benutzers angibt. Folgende Werte sind definiert:

* „pending“ – Das Benutzerkonto wird gerade erstellt. Der Benutzer kann die Benutzerressource nur lesen und bearbeiten (Standardwert).
* Dieser Status ist für einen Benutzer mit "authProvider" als "local" nicht gültig
* "aktiv" - Das Benutzerkonto ist betriebsbereit (Der Benutzer kann sich anmelden, wenn die Kontoeinstellung "isEnabled" "true" ist).
* Dies ist der Standardwert für einen Benutzer mit "authProvider" als "local"
* „gesperrt“ – Das Benutzerkonto ist gesperrt (Der Benutzer kann sich unabhängig von der Kontoeinstellung „isEnabled“ nicht anmelden).




| isEnabled | Zeichenkette | Wahr  a| 
JSON-Zeichenkette, die angibt, ob der Benutzer sich anmelden darf. Definierte Werte sind:

* "true" - Der Benutzer darf sich anmelden; Standardwert bei der Erstellung.
* "false" - Der Benutzer darf sich nicht anmelden.




| authID | Zeichenkette | Falsch  a| 
JSON-Zeichenkette mit der Kennung eines externen Identitätsanbieters. Der Inhalt dieser Zeichenkette hängt vom angegebenen authProvider ab. Für einen Benutzer mit authProvider als „local“ wird „authID“ auf den Wert von „email“ gesetzt. Für einen Benutzer mit authProvider als „ldap“ wird „authID“ auf seinen Distinguished Name (DN) vom LDAP-Server gesetzt.



| authProvider | Zeichenkette | Falsch  a| 
JSON-Zeichenkette, die den Kontext des idAuth Bezeichners angibt. Definierte Werte sind:

* "cloud-central" – Die Authentifizierung erfolgt über den NetApp Cloud Central-Dienst.
* „local“ – Die Authentifizierung erfolgt über den lokalen Benutzer-/Passwortspeicher.
* „ldap“ – Die Authentifizierung des Benutzers erfolgt über einen externen LDAP-Server. Im Astra Control Center müssen alle Benutzer einen Wert von „local“ oder „ldap“ haben. Wird kein Wert angegeben, wird standardmäßig „local“ gewählt. Im Astra Control Service wird „authProvider“ als „local“ oder „ldap“ nicht unterstützt. Wird kein Wert angegeben, wird standardmäßig „cloud-central“ verwendet.




| firstName | Zeichenkette | Wahr  a| 
JSON-Zeichenkette mit einem vom Benutzer angegebenen Vornamen in Langform. Folgende Werte sind definiert:

Ein JSON-Objekt mit einer Sammlung von Feldern, die eine Adresse angeben. Fehlt dieser Schlüssel, wurde keine Adresse angegeben. Existiert dieser Schlüssel, sind alle sechs enthaltenen Felder vorhanden. Wenn vom Benutzer nichts angegeben wurde, ist die Zeichenkette leer.



| sendWelcomeEmail | Zeichenkette | Wahr  a| 
JSON-Zeichenkette, die angibt, ob nach der Validierung eine Willkommens-E-Mail versendet wird. Falls beim Erstellen nicht angegeben, wird der Wert für ACS auf "true" und für Astra Control Center auf "false" gesetzt. Folgende Werte sind definiert:

* "true" - Es wird eine Willkommens-E-Mail versendet; Standardwert bei der Erstellung
* "false" - Es wird keine Willkommens-E-Mail versendet. Für einen Benutzer, bei dem "authProvider" als "local" oder "ldap" angegeben ist, wird dieser Wert ignoriert und auf "false" gesetzt.




| enableTimestamp | Zeichenkette | Falsch  a|

* Entspricht dem ISO-8601 Datums- und Zeitschema




| lastActTimestamp | Zeichenkette | Falsch  a| 
JSON-Zeichenkette mit einem ISO-8601-formatierten Zeitstempel, der angibt, wann die Identity API zuletzt im Namen dieses Benutzers aufgerufen wurde. (Hinweis: Dies ist ressourcenintensiv.) Definierte Werte sind:

* Entspricht dem ISO-8601 Datums- und Zeitschema
